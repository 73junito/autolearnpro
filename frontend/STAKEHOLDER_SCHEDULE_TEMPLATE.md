# Stakeholder Interview — Schedule Template

Purpose: use this template when booking brief discovery interviews (15-30 minutes).

- Meeting length: 20 minutes recommended
- Attendees: Product (owner), Backend owner, Designer, Frontend lead/engineer
- Pre-meeting prep: share `frontend/FRONTEND_REQUIREMENTS.md` and this questionnaire

Agenda (20 minutes)
1. 0:00–0:02 — Quick introductions and meeting goal
2. 0:02–0:08 — Product questions (goals, priorities, acceptance criteria)
3. 0:08–0:14 — Backend/Design specifics (APIs, data shape, design constraints)
4. 0:14–0:17 — Edge cases, performance, security notes
5. 0:17–0:20 — Recap action items and next steps

Post-meeting follow-up
- Capture answers in `frontend/FRONTEND_REQUIREMENTS.md` (API endpoints, sample data)
- Request example JSON payloads and design assets
- Assign follow-up owner for unresolved items

Suggested calendar invite text
"Purpose: 20m discovery to capture requirements for the LMS frontend MVP. Please review `frontend/FRONTEND_REQUIREMENTS.md` and the attached questionnaire before the meeting. We’ll confirm API endpoints, sample data, and design constraints."
